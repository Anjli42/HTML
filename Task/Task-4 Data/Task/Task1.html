Website :

  • A collection of linked web pages that share a unique domain name.

  • A website is also known as a “web presence”.

  • Websites are made of a set of code or programming languages.

  • The most common languages used to build websites are: Hyper text markup language (HTML) helps browsers display webpage elements such as text, links and media. Cascading style sheets (CSS) define webpage styles such as colors, layouts and fonts

  • The first page of a website is called homepage.

  • Types:

    • Static Website : Content & Layout fix. No user interaction. Ex. HTML, CSS, JavaScript.
    • Dynamic Website : Content & Layout change. User interaction. Ex. HTML, CSS, JavaScript, Bootstrap, PHP.

    Webpage :

  • A single document on the web using a unique URL.

  • A web page can be accessed by entering a URL address into a browser's address bar.

  • A web page may contain text, graphics, and hyperlinks to other web pages and files.

  • Types:

    • Static web page
    • Dynamic web page
    • Blog page
    • Contact page
    • Home page
    • Media page

    WWW :

  • It's also known as the web, World Wide Web or W3 or Mesh.

  • Tim Berners-Lee, a British scientist, invented the World Wide Web (WWW).

  • Access the content of websites / webpages from internet.

  • It much easier for people to get information, share, and communicate.

  • 3 protocols of WWW developed by Berners-Lee :

    • Hypertext Markup Language (HTML)
    • Uniform Resource Locator (URL)
    • Hypertext Transfer Protocol (HTTP)

    Protocol :

  • A set of rules for formatting and processing data.

  • Types :

    • HTTP (Hyper Text Transfer Protocol)
    • HTTPS (HyperText Transfer Protocol Secure)
    • FTP (File Transfer Protocol)
    • SMTP (Simple Mail Transfer Protocol)
    • PPP (Point-to-Point Protocol)
    • SFTP (Secure File Transfer Protocol)
    • TCP/IP (Transmission Control Protocol / Internet Protocol)

    HTTP & HTTPS :

  • Internet Protocols

    • HTTP :

  • Hypertext Transfer Protocol

  • Transferring hypermedia documents, such as HTML.

  • The set of rules for transferring files -- such as text, images, sound, video and other multimedia files -- over the web.

  • Resources are exchanged between client devices and servers over the internet.

  • It is a protocol used to access the data on the World Wide Web (WWW).

    • HTTPS :

  • Hypertext Transfer Protocol Secure

  • An extension of the HTTP.

  • A protocol that secures communication and data transfer between a user's web browser and a website.

  • Examples : shopping websites, banking websites, etc.

  • A combination of the HTTP with the Secure Socket Layer (SSL)/Transport Layer Security (TLS) protocol.

    FTP :

  • File transfer protocol.

  • A way to download, upload, and transfer files from one location to another on the Internet and between computer systems

  • It is used to transfer files between a client and a server.

    Client :

  • A device or a machine.

  • Client depend on the services of server, and generates requests for various services.

  • It's runs on the local machine, requesting service from the server.

  • Located on another system or computer, which can be accessed via a network.

  • Examples : smartphones, desktops, laptops, etc.

    Server :

  • A computer program. It's used to provide functionality to other programs.

  • Server authorizes the client's requests and facilitates them with the requested services.

  • Types : web server, application server, database server, cloud server, file server, proxy server, mail server

  • Servers can store and analyze the large data sets.

    Client-Server :

  • Computer networks.

  • The clients requests a resource and the server provides that resource

  • A relationship in which one program (client) requests a service or resource from another program (server).

  • A client does not share any of its resources, but it requests service from a server.

  • Examples: email, network printing, and the World Wide Web.

    URL-URI / URN :

    URI :

  • Uniform Resource Identifier

  • Identify any resource of server.

  • Identify any resource Location or a Name or Both.

  • Types:

    • URL
    • URN

    • URL :

  • Uniform Resource Lactor

  • It's a type (Subset) of URI with Network Location.

  • It's a Not only identifies resource but also include information needed to locate or access it.

  • URL like your home address: it contains all the information to find your home.

  • Resource on the Internet: a web page, an image, a mailbox, etc.

    • URN :

  • Uniform Resource Name

  • It's a type (Subset) of URI with Name within a given space, but no Location.
